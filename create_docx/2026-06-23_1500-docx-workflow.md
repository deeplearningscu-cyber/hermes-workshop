# Docx Generation Workflow Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Provide a seamless workflow to create Word (.docx) documents via Google Docs API when the user issues a request.

**Architecture:** 
- Use Google Workspace OAuth for authentication.
- Leverage Hermes Google Workspace skill (`google-workspace`) for Docs operations.
- Store generated Docs in Google Drive.
- Optionally export as .docx and share.

**Tech Stack:** 
- Python Google API client (fallback)
- Google Docs API via `gws` CLI when available
- Hermes skills: `google-workspace`, `plan`
- Optional: `opencode-env-setup` for dev environment

---

### Task 1: Ensure Google Workspace OAuth is set up
**Objective:** Authenticate Hermes with Google Drive/Docs API.
**Files:** 
- `~/.hermes/google_token.json` (auto-created after setup)
- `~/.hermes/google_client_secret.json` (user-provided)
**Steps:** 
1. Run `python ${HERMES_HOME:-$HOME/.hermes}/skills/productivity/google-workspace/scripts/setup.py --check`.
2. If not authenticated, follow prompts to create OAuth credentials.
3. Provide client secret JSON path when asked.
4. Obtain auth URL, paste back the redirect code.
5. Verify with `setup.py --check` (should print `AUTHENTICATED`).

---

### Task 2: Create a reusable Docs template
**Objective:** Prepare a minimal template document for reuse.
**Files:** 
- `docs/template.docx` (will be stored in Drive as `template.docx` with known ID)
**Steps:** 
1. Use `google_api.py docs create --title "Template"` to create a blank doc.
2. Copy the resulting Doc ID.
3. Store the Doc ID in a config file `~/.hermes/workflows/docx_config.json` as `{ "template_id": "<doc_id>" }`.
4. Optionally add placeholder text like "Title", "Section", etc.

---

### Task 3: Implement `create_docx` command
**Objective:** Provide a CLI command that fills a Docs template with user-supplied content.
**Files:** 
- `scripts/create_docx.py` (executable wrapper)
- `~/.hermes/workflows/docx_config.json` (created in Task 2)
**Steps:** 
1. Read the template ID from config.
2. Load content from user input (passed as arguments or via stdin).
3. Use `google_api.py docs get <template_id>` to fetch body.
4. Append or replace content using `docs append --doc-id <id> --text "<content>"`.
5. Export the filled doc as .docx via Drive download with `--export-mime application/vnd.openxmlformats-officedocument.wordprocessingml.document`.
6. Save the exported file locally under `~/Documents/Generated/<timestamp>.docx`.

---

### Task 4: Wrap everything in a user-friendly CLI
**Objective:** Expose a single entry point `hermes docx generate`.
**Files:** 
- `scripts/hermes_docx_cli.py`
**Steps:** 
1. Parse arguments: `--title`, `--content`, `--output`.
2. Validate that the template exists.
3. Call the script from Task 3 to fill and export.
4. Return success message with file path.

---

### Task 5: Add unit tests
**Objective:** Ensure workflow reliability.
**Files:** 
- `tests/test_docx_generate.py`
**Steps:** 
1. Use `pytest` to run a headless integration test:
   - Create a temporary template doc.
   - Fill it with known content.
   - Verify exported `.docx` matches expected text (use `python-docx` to read).
2. Confirm exit codes: 0 on success, non-zero on failure.
3. Add coverage check: at least 90% of new lines.

---

### Task 6: Documentation & Handoff
**Objective:** Provide clear instructions for end users.
**Files:** 
- `README_DOCX.md` in this workflow folder
**Steps:** 
1. Explain prerequisites (Google OAuth setup).
2. Show example commands:
   ```bash
   hermes docx generate --title "Meeting Notes" --content "Agenda:\n- ..." --output "notes.docx"
   ```
3. List troubleshooting tips (token expired, missing scopes, etc.).
4. Link to related skills: `google-workspace`, `plan`, `opencode-env-setup`.

---

### Risks & Open Questions
- **Scope of templating:** Should we support placeholders like `{{date}}` that get auto-replaced?
- **Concurrency:** Multiple simultaneous `create_docx` calls might overwrite Drive files; need per-user temp naming.
- **Export reliability:** Exporting Google Docs to .docx may alter complex formatting; test with rich templates.
- **Permission model:** Ensure the OAuth scopes cover `https://www.googleapis.com/auth/documents` and `drive`.
